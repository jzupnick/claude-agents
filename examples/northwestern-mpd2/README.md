# Northwestern MPD2 Design Decision System Examples

Real-world applications of the Northwestern MPD2 enhanced design decision system, demonstrating integration across all layers from primitives to project orchestration.

## Example Categories

### Layer Integration Examples
- **`complete-system-integration/`** - Full system usage from concept to delivery
- **`skill-orchestration/`** - Multi-agent coordination through skills
- **`workflow-enhancement/`** - Northwestern framework application to workflows
- **`calculator-usage/`** - Practical application of Northwestern calculators

### Domain Applications
- **`innovation-portfolio/`** - Portfolio management using Three Horizons and Knowledge Funnel
- **`product-development/`** - Systematic product development with stage-gate methodology
- **`strategic-planning/`** - Strategic decision-making with Northwestern frameworks

### Integration Patterns
- **`backward-compatibility/`** - Enhancing existing components without replacement
- **`progressive-enhancement/`** - Migration from simple to complex usage patterns
- **`cross-agent-coordination/`** - Effective multi-agent orchestration

## Example Structure

Each example includes:
- **`problem.md`** - Real problem description and context
- **`approach.md`** - Northwestern methodology application
- **`execution.md`** - Step-by-step implementation with commands
- **`outcomes.md`** - Results and lessons learned
- **`integration.md`** - How components worked together

## Contributing Examples

### Example Creation Guidelines
1. Use real problems you've encountered (not hypothetical)
2. Demonstrate clear value from Northwestern integration
3. Show step-by-step implementation with actual commands
4. Include both successes and challenges encountered
5. Document lessons learned and improvements

### Example Structure Requirements
- Clear problem definition with context
- Northwestern framework selection rationale
- Detailed execution steps with commands
- Actual outputs and results
- Integration analysis and lessons learned

### Quality Standards
- Examples must be reproducible
- Northwestern methodology must be accurately applied
- Integration patterns must be clearly demonstrated
- Learning capture must be comprehensive
- Outcomes must show measurable value
