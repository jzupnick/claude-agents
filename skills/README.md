# Skills

Knowledge domains and orchestration capabilities. This includes both domain expertise that agents and subagents can leverage, and high-level skills that orchestrate multiple agents to accomplish complex, multi-step objectives.

## What Goes Here

### Domain Knowledge Skills
- Technical knowledge (e.g., "kubernetes_patterns", "sql_optimization")
- Design systems (e.g., "design_tokens", "component_library")
- Business logic (e.g., "pricing_models", "user_onboarding")
- Best practices (e.g., "error_handling", "testing_strategies")

### Orchestration Skills
- Multi-agent coordination for complex problems
- Northwestern MPD2 methodology application
- Strategic decision support and portfolio management
- Systematic frameworks for innovation and development

## Format Guidelines

Each skill should be:
- Specific enough to be actionable
- General enough to be reusable
- Opinionated about what works
- Include both positive patterns and anti-patterns

## Integration Strategy

### Orchestration Skills Enhancement
The new Northwestern MPD2 orchestration skills enhance existing capabilities without replacement:

- **Backward Compatible**: All existing domain skills remain functional
- **Enhanced Coordination**: Orchestration skills coordinate existing agents and workflows
- **Academic Rigor**: Northwestern methodologies add systematic frameworks
- **Strategic Context**: Three Horizons and Knowledge Funnel provide decision guidance

### Usage Patterns
- **Simple Problems**: Use individual domain skills directly
- **Complex Coordination**: Use orchestration skills for multi-agent problems
- **Strategic Decisions**: Apply Northwestern frameworks through orchestration skills
- **Systematic Development**: Combine domain skills with orchestration for comprehensive solutions
